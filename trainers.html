Training Pipeline Overview

Core

MiniMind follows a staged training pipeline typical of modern LLMs. Each stage builds on the previous one, progressively refining the model from raw language modelling to aligned, tool-capable agents.

Mathematics — Training Objective Family

Unified View

All objectives can be written as a weighted combination of a supervised term and a regularisation term:

$$\mathcal{L}_{\text{total}} = \underbrace{\alpha\,\mathcal{L}_{\text{supervised}}}_{\text{task loss}} + \underbrace{(1-\alpha)\,\mathcal{L}_{\text{reg}}}_{\text{regularisation}}$$

Where $\mathcal{L}_{\text{reg}}$ is 0 for Pretrain/SFT, a KL penalty for RL stages, and a distillation KL for knowledge transfer.

Stats & Metrics

Throughout the pipeline, you'll encounter various metrics. Generally, Loss represents the error rate (lower is better), Reward represents performance on a specific objective (higher is better), and KL Divergence measures how much the model's behavior is drifting from a safe baseline (lower is better). Monitoring these ensures stable learning.

Loss Function

Learning Rate Schedule

def get_lr(current_step, total_steps, lr):
    cosine = math.cos(math.pi * current_step / total_steps)
    return lr * (0.1 + 0.45 * (1 + cosine))

Mathematics

Pretrain Objective
$$\mathcal{L}_{\text{pretrain}} = -\frac{1}{T}\sum_{t=1}^{T}\log P_\theta(x_t \mid x_1,\ldots,x_{t-1})$$ $$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{CE}} + \lambda_{\text{aux}}\,\mathcal{L}_{\text{aux}}$$

The cosine learning-rate schedule anneals from peak $\eta_{\max}$ to $0.1\,\eta_{\max}$:

$$\eta_t = \eta_{\max}\!\left(0.1 + 0.45\!\left(1 + \cos\!\frac{\pi t}{T}\right)\right)$$

Training Workflow

LoRA Module

class LoRA(nn.Module):
    def __init__(self, in_features, out_features, rank=16):
        self.A = nn.Linear(in_features, rank, bias=False)  # std=0.02
        self.B = nn.Linear(rank, out_features, bias=False) # zero init

    def forward(self, x):
        return self.B(self.A(x))

Mathematics

LoRA Forward & Merge
$$h = \underbrace{W_0 x}_{\text{frozen base}} + \underbrace{B A x}_{\text{LoRA delta}}, \quad A\in\mathbb{R}^{r\times d},\; B\in\mathbb{R}^{d\times r},\; r \ll d$$

Trainable parameters: $2 \cdot r \cdot d$ vs. $d^2$ for full fine-tuning. Compression ratio: $\frac{2r}{d}$.

$$W_{\text{merged}} = W_0 + B A \quad \text{(at inference — no overhead)}$$

Parameter efficiency: for $d=768, r=16$ we train $\approx 0.8\%$ of layer parameters.

LoRA training tracks the same Masked CE Loss as SFT. Because LoRA trains less than 1% of the model's parameters, the loss curve might look noisier or decrease slightly slower than full fine-tuning. However, it prevents catastrophic forgetting of the base model's knowledge.

Loss Computation

Loss Function

def dpo_loss(ref_log_probs, policy_log_probs, mask, beta):
    ref = (ref_log_probs * mask).sum(dim=1)
    pol = (policy_log_probs * mask).sum(dim=1)
    half = ref.shape[0] // 2

    pi_logratios  = pol[:half] - pol[half:]   # chosen − rejected (policy)
    ref_logratios = ref[:half] - ref[half:]   # chosen − rejected (ref)

    logits = pi_logratios - ref_logratios
    return -F.logsigmoid(beta * logits).mean()

Mathematics

DPO Objective
$$\mathcal{L}_{\text{DPO}} = -\mathbb{E}_{(x,y_w,y_l)}\!\left[\log\sigma\!\left(\beta\underbrace{\left(\log\frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \log\frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right)}_{\text{implicit reward gap}}\right)\right]$$

$\beta$ controls the KL penalty strength — large $\beta$ keeps the policy close to the reference; small $\beta$ allows more deviation toward preferences.

DPO's implicit reward: $r_\phi(x,y)=\beta\log\frac{\pi_\theta(y|x)}{\pi_{\text{ref}}(y|x)}$

  • Chosen/Rejected Log-Probs: Log-probabilities of chosen responses should increase, while rejected responses decrease.

GAE Advantage

Critic Model

class CriticModel(MiniMindForCausalLM):
    def __init__(self, params):
        super().__init__(params)
        self.value_head = nn.Linear(params.hidden_size, 1)  # replaces lm_head

    def forward(self, input_ids, attention_mask=None):
        outputs = self.model(input_ids, attention_mask=attention_mask)
        hidden  = self.model.norm(outputs[0])
        return self.value_head(hidden).squeeze(-1)  # per-token values

Mathematics

PPO Objectives
$$r_t(\theta) = \frac{\pi_\theta(a_t|s_t)}{\pi_{\theta_{\text{old}}}(a_t|s_t)}$$ $$\mathcal{L}^{\text{CLIP}}(\theta) = -\mathbb{E}_t\!\left[\min\!\left(r_t\hat{A}_t,\;\operatorname{clip}(r_t,1\!-\!\varepsilon,1\!+\!\varepsilon)\hat{A}_t\right)\right]$$ $$\mathcal{L}^{\text{KL}} = \beta\,\mathbb{E}_t\!\left[\exp(\log\pi_{\text{ref}} - \log\pi_\theta) - (\log\pi_{\text{ref}} - \log\pi_\theta) - 1\right]$$ $$\mathcal{L}^{\text{VF}}(\psi) = \tfrac{1}{2}\,\mathbb{E}_t\!\left[\max\!\left((V_\psi - R_t)^2,\;(\operatorname{clip}(V_\psi, V_{\text{old}}\pm\varepsilon) - R_t)^2\right)\right]$$ $$\mathcal{L}^{\text{PPO}} = \mathcal{L}^{\text{CLIP}} + \mathcal{L}^{\text{KL}} + c\,\mathcal{L}^{\text{VF}}$$

GAE advantage: $\hat{A}_t = \sum_{l=0}^{\infty}(\gamma\lambda)^l\,\delta_{t+l}$, where $\delta_t = r_t + \gamma V_{t+1} - V_t$.

Stats & Metrics

PPO involves multiple moving parts, so you monitor a suite of metrics in wandb/tensorboard:

  • Actor Loss: The policy objective (clipped surrogate).
  • Critic/Value Loss: How accurately the value model predicts the reward.
  • KL Divergence: Measures how far the policy drifts from the reference model. High KL means the model is "reward hacking" and losing language capability.
  • Reward: The average score from the Reward Model (should rise steadily).
Graph Explanation:
• X-axis (Step): PPO rollout & update steps.
• Y-axis (Values): A composite of multiple metrics. Typically, Actor Loss oscillates near zero, Critic Loss drops as the value network gets better at predicting rewards, and Reward climbs upwards.
• Trend: You will see the total Reward line steadily increasing as the model learns to generate answers that the Reward Model prefers. If KL divergence shoots up rapidly, the model is "hacking" the reward and losing text quality.

CISPO vs GRPO

Loss — Two Variants

kl_div = ref_logps - logps
per_token_kl = torch.exp(kl_div) - kl_div - 1
ratio = torch.exp(logps - old_logps)

# ── CISPO variant ──
clamped_ratio = torch.clamp(ratio, max=epsilon_high).detach()
per_token_loss = -(clamped_ratio * adv.unsqueeze(1) * logps - beta * per_token_kl)

# ── GRPO variant ──
clipped_ratio = torch.clamp(ratio, 1-ε, 1+ε)
per_token_loss = -(torch.min(ratio * adv.unsqueeze(1),
                              clipped_ratio * adv.unsqueeze(1)) - beta * per_token_kl)

Mathematics

GRPO & CISPO Objectives

Group advantage normalisation (μ, σ computed within each group of N outputs):

$$\hat{A}_i = \frac{r_i - \mu_{\text{group}}}{\sigma_{\text{group}} + 10^{-4}}$$

GRPO loss (PPO-style clip):

$$\mathcal{L}_{\text{GRPO}} = -\mathbb{E}\!\left[\min\!\left(r(\theta)\hat{A},\;\operatorname{clip}(r(\theta),1\!-\!\varepsilon,1\!+\!\varepsilon)\hat{A}\right)\right] + \beta\,D_{\text{KL}}(\pi_{\text{ref}}\|\pi_\theta)$$

CISPO loss (upper-clamp only — avoids reward hacking):

$$\mathcal{L}_{\text{CISPO}} = -\mathbb{E}\!\left[\min(r(\theta),\varepsilon_{\text{high}})\cdot\hat{A}\cdot\log\pi_\theta\right] + \beta\,D_{\text{KL}}(\pi_{\text{ref}}\|\pi_\theta)$$

Temperature Softening

Distillation Loss

def distillation_loss(student_logits, teacher_logits, temperature=1.0):
    with torch.no_grad():
        teacher_probs = F.softmax(teacher_logits / temperature, dim=-1)
    student_log_probs = F.log_softmax(student_logits / temperature, dim=-1)
    kl = F.kl_div(student_log_probs, teacher_probs, reduction='batchmean')
    return (temperature ** 2) * kl

Mathematics

Knowledge Distillation Objective

Soft targets via temperature $T$:

$$p_i^T = \frac{\exp(z_i^T / T)}{\sum_j\exp(z_j^T/T)}, \quad q_i^T = \frac{\exp(z_i^S/T)}{\sum_j\exp(z_j^S/T)}$$

Combined loss with $T^2$ rescaling (preserves gradient magnitude):

$$\mathcal{L} = \alpha\cdot\underbrace{\text{CE}(y,\pi_S)}_{\text{hard label}} + (1-\alpha)\cdot T^2\cdot\underbrace{D_{\text{KL}}\!\left(\sigma\!\left(\tfrac{z_T}{T}\right)\Big\|\sigma\!\left(\tfrac{z_S}{T}\right)\right)}_{\text{soft knowledge transfer}}$$

The $T^2$ factor compensates for the reduced gradient scale when using soft targets, keeping the two loss terms in similar magnitude regardless of temperature.

In Distillation, you monitor two competing loss terms:

  • Cross-Entropy (Hard Loss): How well the student predicts the actual ground-truth tokens.
  • KL Divergence (Soft Loss): How closely the student's probability distribution matches the teacher's softened distribution.

Both should decrease, indicating the student is absorbing both factual knowledge (hard) and reasoning nuance (soft).

BPE greedily merges the pair $(a, b)$ with the highest co-occurrence frequency at each step:

$$\text{merge}^* = \arg\max_{(a,b)}\;\text{count}(a,\,b\;\text{adjacent in corpus})$$

Mathematics

LoRA Forward & Merge Mathematics

During training (base frozen, only A and B updated):

$$h = W_0 x + \Delta W x = W_0 x + B A x$$

Initialisation — ensures $\Delta W = 0$ at step 0:

$$A \sim \mathcal{N}(0, \sigma^2),\quad B = \mathbf{0}$$

After merge (zero inference overhead):

$$W_{\text{merged}} = W_0 + B A \in \mathbb{R}^{d\times d}$$

Parameter savings for a single $d\times d$ layer with rank $r$:

$$\frac{|\text{LoRA params}|}{|\text{full params}|} = \frac{2rd}{d^2} = \frac{2r}{d}$$

For $d=768,\, r=16$: trains only $\mathbf{4.2\%}$ of parameters per layer.

During training: $h = W_{\text{base}}x + BAx$  ·  After merge: $h = (W_{\text{base}} + BA)x$
