Training Pipeline Overview

Core

MiniMind follows a staged training pipeline typical of modern LLMs. Each stage builds on the previous one:

    B --> C["DPO

    B --> D["PPO/GRPO

    B --> E["Agentic RL
train_agent.py"]
    B --> F["Distillation
train_distillation.py"]
    B --> G["LoRA

    style B fill:#23863633,stroke:#3fb950
    style C,D,E fill:#9e6a0333,stroke:#d29922
    style F,G fill:#bc8cff33,stroke:#bc8cff

Pipeline Flow

  1. Pretrain — learns language patterns & world knowledge via next-token prediction on raw text.
  2. Full SFT — instruction-following on assistant responses only (prompt tokens masked).
  3. RLHF/DPO — aligns model to human preferences using chosen/rejected pairs.
  4. RLAIF (PPO/GRPO) — optimizes generation via reinforcement learning with a reward model.
  5. Agentic RL — multi-turn tool-use training with GRPO/CISPO and ground-truth verification.
  6. Distillation — compresses a larger teacher model into a smaller student using KL divergence.
  7. LoRA — parameter-efficient fine-tuning via low-rank adapters.

train_pretrain.py

Pretrain

Goal

Autoregressive next-token prediction on raw text corpus. Builds the model's base knowledge of language patterns, facts, and syntax.

Data Format

JSONL with a single "text" field. Loaded via PretrainDataset which appends BOS/EOS tokens and pads to max_seq_len (default 340).

{"text": "Transformer通过自注意力机制建模上下文关系，是现代大语言模型的重要基础结构。"} {"text": "机器学习是人工智能的核心技术之一，通过算法让计算机从数据中学习规律。"}

Loss Function

The model's built-in cross-entropy over the entire sequence (no masking — every token is trained).

# Inside training loop
res = model(input_ids, labels=labels)

Learning Rate Schedule

def get_lr(current_step, total_steps, lr):
    cosine = math.cos(math.pi * current_step / total_steps)
    return lr * (0.1 + 0.45 * (1 + cosine))

train_lora.py

LoRA

Goal

Parameter-efficient fine-tuning. Only trains low-rank adapters (matrices A and B) injected into attention projections. Base model is frozen.

Architecture

(Gaussian init)"]

LoRA Module (model_lora.py)

class LoRA(nn.Module):
    def __init__(self, in_features, out_features, rank=16):
        self.A = nn.Linear(in_features, rank, bias=False)  # std=0.02
        self.B = nn.Linear(rank, out_features, bias=False) # zero init

    def forward(self, x):
        return self.B(self.A(x))

Forward pass modification

apply_lora() monkey-patches every nn.Linear where in_features == out_features:

def forward_with_lora(x, original_fwd, lora):
    return original_fwd(x) + lora(x)

Math

h = W₀x + ΔWx = W₀x + BAx   where   A ∈ ℝrank×d, B ∈ ℝd×rank, rank ≪ d

RLHF

Goal

Direct Preference Optimization — align the model to prefer chosen responses over rejected ones using a reference model as a KL anchor.

Architecture Flow

Loss Function

def dpo_loss(ref_log_probs, policy_log_probs, mask, beta):
    ref = (ref_log_probs * mask).sum(dim=1)
    pol = (policy_log_probs * mask).sum(dim=1)
    half = ref.shape[0] // 2

    pi_logratios  = pol[:half] - pol[half:]       # chosen - rejected (policy)
    ref_logratios = ref[:half] - ref[half:]       # chosen - rejected (ref)

    logits = pi_logratios - ref_logratios
    return -F.logsigmoid(beta * logits).mean()

Math

ℒDPO = -𝔼(x,yw,yl) [ log σ( β ( log πθ(yw|x) - log πθ(yl|x) - (log πref(yw|x) - log πref(yl|x))) ) ]

Critic Model

class CriticModel(MiniMindForCausalLM):
    def __init__(self, params):
        super().__init__(params)
        self.value_head = nn.Linear(params.hidden_size, 1)  # replaces lm_head

    def forward(self, input_ids, attention_mask=None):
        outputs = self.model(input_ids, attention_mask=attention_mask)
        hidden = self.model.norm(outputs[0])
        return self.value_head(hidden).squeeze(-1)  # per-token values

GAE Advantage Calculation

lastgaelam = 0
advs_rev = []
for t in reversed(range(gen_len)):
    nv = values[:, t+1] if t < gen_len-1 else 0
    delta = rewards[:, t] + gamma * nv - values[:, t]
    lastgaelam = delta + gamma * lam * lastgaelam
    advs_rev.append(lastgaelam)
advantages = torch.stack(advs_rev[::-1], dim=1)
returns = advantages + values

Actor Loss (clipped surrogate + KL penalty)

ratio = torch.exp(log_ratio)

# KL penalty: exp(kl) - kl - 1
kl_penalty = ((torch.exp(ref_logp - logp) - (ref_logp - logp) - 1.0)
              * mask).sum() / mask.sum().clamp(min=1)

policy_loss = (torch.max(
    -adv * ratio,
    -adv * torch.clamp(ratio, 1.0-ε, 1.0+ε)
) * mask).sum() / mask.sum().clamp(min=1) + kl_coef * kl_penalty

Value Loss (clipped MSE)

value_loss = 0.5 * (torch.max(
    (V - returns) ** 2,
    (clip(V, old_V-ε, old_V+ε) - returns) ** 2
) * mask).sum() / mask.sum().clamp(min=1)

ℒPPO(θ) = -𝔼[min(rt(θ)Ât, clip(rt(θ), 1-ε, 1+ε)Ât)] + β · KL(πref ‖ πθ)
ℒVF(ψ) = ½𝔼[max((Vψ - Rt)², (clip(Vψ, Vold±ε) - Rt)²)]

Group Advantage Calculation

grouped = rewards.view(-1, num_generations)   # [B, num_gen]
mean_r = grouped.mean(dim=1).repeat_interleave(num_generations)
std_r  = grouped.std(dim=1, unbiased=False).repeat_interleave(num_generations)
advantages = (rewards - mean_r) / (std_r + 1e-4)

Loss — Two Variants

kl_div = ref_logps - logps
per_token_kl = torch.exp(kl_div) - kl_div - 1
ratio = torch.exp(logps - old_logps)

# ---- CISPO variant ----
clamped_ratio = torch.clamp(ratio, max=epsilon_high).detach()
per_token_loss = -(clamped_ratio * adv.unsqueeze(1) * logps - beta * per_token_kl)

# ---- GRPO variant (standard PPO-style) ----
clipped_ratio = torch.clamp(ratio, 1-ε, 1+ε)
loss1 = ratio * adv.unsqueeze(1)
loss2 = clipped_ratio * adv.unsqueeze(1)
per_token_loss = -(torch.min(loss1, loss2) - beta * per_token_kl)

train_agent.py

Agentic RL

Goal

Multi-turn tool-use training via RL. The agent calls tools (<tool_call> tags), gets results, and continues — all within a GRPO/CISPO loss framework.

Data Format

{ "conversations": [ {"role": "system", "content": "# Tools\n\nYou have access to...", "tools": "[{\"type\":\"function\",\"function\":{\"name\":\"calculate_math\",...}}]"}, {"role": "user", "content": "帮我算一下256*37"}, {"role": "assistant", "content": "", "tool_calls": "[{\"name\":\"calculate_math\",\"arguments\":{\"expression\":\"256*37\"}}]"}, {"role": "tool", "content": "{\"result\":\"9472\"}"}, {"role": "assistant", "content": "结果是9472。"} ], "gt": ["9472"]}

Reward Calculation

Composite reward with multiple signals:

def calculate_rewards(responses, gt_batch, reward_model, ...):
    # No tool calls → length + thinking + RM score + repetition
    # With tool calls → tool alignment + GT verification + RM + repetition

    # Tool alignment: how many tool calls match valid tool definitions
    tool_gap = |valid_calls| - |gt| + excess_invalid

    # GT verification: what fraction of expected answers appear in text
    verified = validate_gt_in_text(response, ground_truth)
    gt_score = 2.5 * |verified| / |gt|

    # Final clipped to [-3, 3]
    reward = min(max(reward, -3.0), 3.0)

Multi-turn Rollout

def rollout_single(engine, tokenizer, messages, tools, max_turns=3):
    for turn in range(max_turns):
        context = tokenizer.apply_chat_template(messages, tools=tools, ...)
        result = engine.rollout(context, ...)
        calls = parse_tool_calls(result.text)
        if not calls: break
        for call in calls:
            result = execute_tool(call["name"], call["arguments"])
            messages.append({"role": "tool", "content": json.dumps(result)})

train_distillation.py

Distillation

Goal

Knowledge Distillation: train a smaller student model to match both the ground-truth labels and the softened output distribution of a larger teacher model.

Architecture

graph TD
    X["Input tokens"] --> S["Student Model
(trainable)"]
    X --> T["Teacher Model
(frozen)"]
    S --> CE["Cross-Entropy
vs ground-truth"]
    T --> KL["KL Divergence
(softened)"]
    S --> KL
    CE --> L["α·CE + (1-α)·KL"]
    KL --> L

Distillation Loss

def distillation_loss(student_logits, teacher_logits, temperature=1.0):
    with torch.no_grad():
        teacher_probs = F.softmax(teacher_logits / temperature, dim=-1)

    student_log_probs = F.log_softmax(student_logits / temperature, dim=-1)
    kl = F.kl_div(student_log_probs, teacher_probs, reduction='batchmean')
    return (temperature ** 2) * kl

Total Loss

# CE on assistant tokens only
ce_loss = F.cross_entropy(student_logits, shift_labels, ignore_index=-100)

# KL on assistant tokens only
distill_loss = distillation_loss(student_logits[mask], teacher_logits[mask], temp)

# Combined
loss = α * ce_loss + (1 - α) * distill_loss

Math

ℒ = α · CE(y, πS) + (1-α) · T² · KL(softmax(πT/T) ‖ softmax(πS/T))

Datasets (lm_dataset.py)

Data

PretrainDataset

Loads JSONL with {"text": "..."}. Tokenizes with BOS+EOS, pads to max_length. Labels = input_ids with padding masked to -100.

SFTDataset

Loads JSONL with {"conversations": [...]}. Key features:

  • create_chat_prompt() — uses tokenizer.apply_chat_template() with tools support
  • generate_labels() — masks everything before assistant\n response as -100
  • pre_processing_chat() — 20% chance to add random system prompt
  • post_processing_chat() — 80% chance to strip empty <think> tags
  • Supports tool_calls and reasoning_content fields

DPODataset

Loads JSONL with {"chosen": [...], "rejected": [...]}. Returns dict with x_chosen, y_chosen, mask_chosen, x_rejected, y_rejected, mask_rejected.

RLAIFDataset

Loads JSONL with {"conversations": [...]}. Returns {"prompt": str, "answer": ""} with optional thinking tags based on thinking_ratio.

AgentRLDataset

Loads JSONL with {"conversations": [...], "gt": [...]}. Extracts multi-turn messages, parses tools from system messages, returns {"messages": [...], "tools": {...}, "gt": [...]}.
